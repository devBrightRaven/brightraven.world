Brand Style Guide

A quiet system for story-led tools.

This page is the local reference for keeping Bright Raven pages visually consistent: color tokens, type roles, hover states, component patterns, and multilingual routing.

Design Principles

Story before inventory

Start with a recognizable human moment. Introduce projects after the visitor understands why they matter.

Subject before system

Interfaces should help people act without making the system feel like the main character.

Visible, not overpromised

Lab instruments can be public while still being clearly marked as prototypes, studies, or early maps.

Multilingual entry

Use separate pages for English, Traditional Chinese, Simplified Chinese, and Japanese. Do not stack four languages inside one page.

Typography

Display

You opened the library again.

Use Cormorant Garamond for large English story headings. CJK pages fall back to the platform serif stack for now; add dedicated Noto Serif CJK fonts only if the design needs stronger typographic control.

Body / UI

Inter and the platform sans-serif stack carry navigation, body copy, status labels, buttons, and product explanations. Keep line-height generous and avoid dense academic blocks on entry pages.

Usage Rules

Do

  • Use real product screenshots when a page explains a tool.
  • Start entry pages with a human scene, then name the concept.
  • Mark Lab work by maturity: prototype, design study, early map.
  • Keep hover and focus states visible enough to notice immediately.

Do Not

  • Do not make every page look like a product directory.
  • Do not hide unfinished work, but do not present it as a finished product.
  • Do not use decorative gradients or atmospheric images when a real screenshot can explain the thing.
  • Do not mix four languages in one content block.
